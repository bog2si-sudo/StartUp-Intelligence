# 24B_OPENAI_REPORT_TEMPLATE

Version: 4.0

Status: Production

------------------------------------------------------------------------

# Purpose

This document defines the narrative template that the OpenAI model must
populate.

The application controls:

-   report sections,
-   scores,
-   calculations,
-   benchmark selection,
-   business logic.

The language model generates only the narrative for each section.

------------------------------------------------------------------------

# Report Structure

The generated output must contain the following sections in this exact
order:

1.  Executive Summary
2.  Current Position
3.  Evidence Assessment
4.  Benchmark Context
5.  Key Strengths
6.  Primary Risks
7.  Evidence Gaps
8.  Priority Actions
9.  Decision Readiness
10. 30-Day Roadmap
11. 90-Day Roadmap
12. Closing Assessment

No additional sections may be introduced.

------------------------------------------------------------------------

# Section Requirements

## Executive Summary

Summarise:

-   current readiness,
-   strongest evidence,
-   main execution constraint,
-   recommended immediate focus.

Maximum length: 250 words.

------------------------------------------------------------------------

## Current Position

Explain the startup's current stage using only supplied evidence.

Do not speculate.

------------------------------------------------------------------------

## Evidence Assessment

Describe:

-   strongest evidence,
-   weakest evidence,
-   overall evidence quality,
-   confidence level.

Reference only deterministic outputs supplied by the application.

------------------------------------------------------------------------

## Benchmark Context

Explain why selected benchmark records are relevant.

Do not introduce benchmark information not supplied by the application.

------------------------------------------------------------------------

## Key Strengths

Describe validated strengths supported by founder responses.

------------------------------------------------------------------------

## Primary Risks

Describe the highest-impact execution risks.

Each risk must reference observable evidence or missing evidence.

------------------------------------------------------------------------

## Evidence Gaps

List the most important missing information preventing greater
confidence.

------------------------------------------------------------------------

## Priority Actions

Generate narrative for the prioritised actions already selected by the
application.

Do not reprioritise them.

------------------------------------------------------------------------

## Decision Readiness

Explain whether the startup appears ready for its next major decision
based on available evidence.

------------------------------------------------------------------------

## Roadmaps

Produce concise narrative for:

-   first 30 days,
-   first 90 days.

Recommendations must align with the deterministic priority list.

------------------------------------------------------------------------

## Closing Assessment

Provide an objective conclusion.

Do not predict future success.

Reinforce that conclusions are based on available evidence.

------------------------------------------------------------------------

End of document.
