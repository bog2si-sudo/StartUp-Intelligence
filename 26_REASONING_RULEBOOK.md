# 26_REASONING_RULEBOOK

Version: 4.0

## Purpose

Define deterministic business reasoning.

## Rules

1. Identify exactly one primary constraint.
2. Use founder evidence before benchmark evidence.
3. Missing evidence lowers confidence, not quality.
4. Recommendations must improve evidence, confidence or value creation.
5. Rank recommendations by:
   - Impact
   - Confidence
   - Dependencies
   - Reversibility
   - Cost
   - Effort
6. The LLM never creates business logic.
7. Every recommendation must include:
   - Why now
   - Evidence
   - Success criteria
