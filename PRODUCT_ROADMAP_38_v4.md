# 38_PRODUCT_ROADMAP

Version: 4.0

Status: Production

------------------------------------------------------------------------

# Purpose

This document defines the planned evolution of the Founder Intelligence
Platform.

The roadmap prioritises incremental delivery while preserving
deterministic business logic and architectural simplicity.

------------------------------------------------------------------------

# Version 4.0 (Current)

Scope:

-   Three Tally forms
-   Stripe payment inside Tally
-   Supabase persistence
-   Modular OpenAI reasoning pipeline
-   Markdown report generation
-   Private report pages
-   Resend email delivery

Objective:

Deliver a stable production-ready MVP.

------------------------------------------------------------------------

# Version 4.1

Focus:

-   Operational improvements
-   Better monitoring
-   Enhanced reporting
-   Improved benchmark management

No architectural redesign.

------------------------------------------------------------------------

# Version 4.2

Focus:

-   Expanded benchmark dataset
-   Additional report insights
-   Improved internal tooling
-   Administrative workflow enhancements

Business rules remain deterministic.

------------------------------------------------------------------------

# Version 5.0

Potential future capabilities:

-   User authentication
-   Founder dashboard
-   Assessment history
-   Benchmark administration
-   Multi-language support

These items remain outside the MVP scope and should not delay Version 4.

------------------------------------------------------------------------

# Roadmap Principles

Every future enhancement should:

-   preserve deterministic processing;
-   minimise operational complexity;
-   remain compatible with the existing architecture;
-   avoid unnecessary technology additions.

------------------------------------------------------------------------

End of document.
