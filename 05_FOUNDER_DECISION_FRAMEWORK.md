# 05_FOUNDER_DECISION_FRAMEWORK

Version: 4.0

## Purpose

Describe how the platform makes decisions.

## Core Principle

The objective is not to improve everything.

The objective is to identify today's primary constraint.

## Decision Flow

Evidence
↓
Interpretation
↓
Constraint
↓
Recommendation
↓
Value Creation

## Resource Allocation

Recommendations compete for:
- Founder time
- Capital
- People
- Management attention

## Recommendation Order

1. Expected impact
2. Confidence
3. Dependencies
4. Reversibility
5. Cost
6. Effort

## Rules

- One primary constraint.
- Maximum two secondary constraints.
- Every recommendation must reference evidence.
- Missing evidence lowers confidence, not quality.
